Zia RAG Bot - Technical Architecture

System Architecture

Hybrid RAG system combining BM25 keyword search, Hnswlib vector embeddings, and Reciprocal Rank Fusion for optimal retrieval accuracy.

Hybrid RAG Pipeline

BM25 Keyword Search

Library: rank_bm25 (BM25Okapi)
Preprocessing: NLTK tokenization, stopword removal, lowercasing
Scoring: TF-IDF based relevance scoring
Threshold: Score > 1.0 for relevance
Use Case: Exact keyword matching for specific queries

Hnswlib Vector Search

Library: hnswlib (Hierarchical Navigable Small World)
Embeddings: Google Gemini text-embedding-004 (768 dimensions)
Index: Cosine similarity space
Parameters: ef_construction=200, M=16, ef=50
Batch Processing: 10 texts per embedding batch
Use Case: Semantic similarity for conceptual queries

Reciprocal Rank Fusion (RRF)

Algorithm: Combines BM25 and vector search results
Formula: score = 1/(k + rank), k=60
Process: Merge results from both methods, sum scores, re-rank
Output: Top-5 most relevant documents
Benefit: Balances keyword precision with semantic understanding

Supervised Learning Layer

Storage: supervised_learning.json
Matching: BM25 on query variations
Threshold: Score > 1.0 for supervised match
Priority: Supervised responses override RAG search
Training: Admin can add query-response pairs with auto-generated variations

Database Schema

conversation_history Table

Fields: id, session_id, conversation_id, user_input, bot_response, timestamp
Indexes: session_id, conversation_id
Purpose: Store all chat messages grouped by conversation
Limit: MAX_MEMORY = 25 messages per query context

feedback Table

Fields: id, session_id, user_input, bot_response, feedback_text, feedback_type, timestamp
Purpose: Collect user feedback (like/dislike) for model improvement
Types: Positive, negative, custom text feedback

Session Management

Session ID: UUID per browser (Flask session)
Conversation ID: UUID per chat thread
Isolation: Per-browser conversation history
Cleanup: Automatic deletion of entries older than 24 hours
Persistence: 7-day session lifetime

Arbor - Technical Architecture

System Architecture

Dual-mode research engine with real-time web scraping, DuckDuckGo search, and mandatory citation grounding using Gemini 2.5 Flash.

Dual-Mode Research System

Quick Search Mode

Speed: <3s Time-to-First-Token
Sources: 10 search results
Scraping: Top 3 URLs, 2000 chars/page
Output: Concise answer with inline citations
Use Case: Rapid fact-retrieval and quick lookups

Deep Research Mode

Depth: Comprehensive analysis
Sources: 25 search results
Scraping: Up to 10+ dense sources, 15000 chars/page
Output: 2500-4000 word structured report
Sections: Executive Summary, Introduction, Key Findings, Analysis, Recommendations, Conclusion
Use Case: In-depth research and comprehensive reports

Chunked Generation

Deep research uses chunked generation to avoid token limits. Part 1: Executive Summary + Introduction + Key Findings (first half). Part 2: Key Findings (second half) + Analysis + Recommendations + Conclusion. Fallback strategy ensures complete reports even if single generation fails.

Anti-Hallucination System

Grounding Mechanism

Every claim must be grounded in scraped sources. Inline citations [N] format mapped to source indices. Gemini instructed to cite specific sources. Responses without citations are rejected.

Chain-of-Thought Reasoning

Mandatory <thinking> block before response. AI plans search strategy and analyzes sources. Extracted and logged for debugging. Improves response quality and reduces hallucinations.

Source Verification

URL accessibility verification. Content extraction with Trafilatura + BeautifulSoup fallback. User-Agent rotation to avoid blocking. 8-second timeout per request. Polite 0.3s delay between requests.

Quality Metrics

Hallucination Reduction: ~90% vs raw zero-shot prompting
Citation Accuracy: 95%+ (inline markers match sources)
Source Diversity: Multi-source aggregation
Freshness: Real-time web scraping

Web Scraping Pipeline

DuckDuckGo Search

Library: duckduckgo_search (DDGS)
Region: India (in-en) for better relevance
Retry Logic: 3 attempts with 1s delays
Caching: 5-minute TTL to reduce API calls
Fallback: Wikipedia URL if search fails

Content Extraction

Primary: Trafilatura (optimized for article extraction)
Fallback: BeautifulSoup4 with intelligent container detection
Containers: article, main, #content, .content, .entry-content
Filtering: Remove scripts, styles, nav, footer
Limits: 15KB per page (Quick), 15KB per page (Deep)

Anti-Blocking Measures

6 rotating User-Agents (Chrome, Firefox, Edge, Safari). Browser-like headers (Accept, Accept-Language, DNT). Connection keep-alive. Polite crawling with delays. Timeout handling (8s per request).
